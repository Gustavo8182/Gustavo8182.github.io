Se você ainda hospeda as suas aplicações frontend em servidores web como apache ou nginx, sinto dizer mas você provavelmente está perdendo dinheiro.

Quando se fala em serverless, sempre pensamos no Lambda da AWS, no Google Cloud Functions, App Engine, etc. até surgir a seguinte dúvida: Se são arquivos estáticos, pra a gente precisa subir um servidor?

No começo, ferramentas como S3 e Google Cloud Storage só serviam para armazenar e compartilhar arquivos. Conforme a tecnologia na nuvem foi evoluindo, esses serviços se tornaram muito úteis e poderosos, permitindo que possamos usá-los para hospedar uma aplicação de página única, sem precisar de servidores.

Neste artigo, usaremos o exemplo "hello world" do Angular para criar nossa aplicação. Antes de começar, você precisa ter a versão LTS do Node.js instalada e instalar o Angular CLI com o comando npm install -g @angular/cli.

Com o Node.js e o Angular CLI instalados, basta executar o comando ng new my-s3-app para criar seu aplicativo. Ele fará duas perguntas:

  • Would you like to add Angular routing? (y/N)
  • Which stylesheet format would you like to use?

Na prática, a escolha aqui não faz muita diferença. Eu geralmente escolho "yes" para adicionar o roteamento (digite "y" e pressione Enter) e seleciono o SCSS para os estilos (use as setas e pressione Enter).

Após isso, o Angular criará a aplicação e instalará as dependências necessárias para o seu projeto ser executado. Abra a pasta do projeto no terminal e digite ng serve para executar o servidor localmente e ver se está tudo certo. Acesse localhost:4200 no navegador e você verá a página inicial.

No terminal, digite o comando "ng build". Isso criará uma pasta chamada "dist" dentro do seu projeto com a versão de produção do seu aplicativo. Lembre-se dessa pasta, pois são os arquivos dela que serão enviados para o S3.

Vamos publicar a nossa aplicação no Amazon S3. Para isso, é necessário ter uma conta AWS. Se você não tiver uma, pode se cadastrar em https://aws.amazon.com. No primeiro ano, você pode experimentar vários serviços gratuitamente. É uma ótima oportunidade para testar diferentes configurações e serviços.

Para criar o Bucket, faça login no console AWS e selecione o menu Services > Storage > S3. Lá, precisamos criar um bucket para armazenar a aplicação. O nome do bucket é único em todas as contas da AWS e fará parte do nome de domínio, então escolha um nome que ainda não tenha sido usado, mas que não seja estranho.

Dependendo da região, o S3 pode usar dois formatos de URL:

s3-website dash (-) Region ‐ http://nomedobucket.s3-website-Region.amazonaws.com

ps3-website dot (.) Region ‐ http://nomedobucket.s3-website.Region.amazonaws.com

Subindo os arquivos

Clique no bucket que acabamos de criar, depois clique em upload e depois em add files. Vá para a pasta dist my-s3-app que o ng build acabou de gerar e selecione todos os arquivos usando o mouse, ctrl+a (windows) . Você deve ver esse resultado:

Se tudo estiver certo, basta clicar em upload, esperar a confirmação e partir para a próxima etapa.

Ativando a hospedagem:

Agora, para ativarmos a hospedagem de sites estáticos, vá em Properties > Static website hosting > Edit.

Na página de edição, selecione "enable" para habilitar a hospedagem de sites estáticos. Preencha "index.html" nos campos Index e Error document e salve as alterações.

Ao voltar na aba properties, uma URL terá sido criada!

Porém, ao tentar acessar, você verá a seguinte mensagem:

Isso acontece porque todos os buckets são privados por padrão, tornando-os inacessíveis.

Para alterar isso, ainda no Bucket, vamos precisar ir na aba Permissions > Block public access > Edit.

Nessa tela, desmarcamos a opção Block all public access e deixamos marcado somente as opções:

  • Block public access to buckets and objects granted through new access control lists (ACLs)
  • Block public access to buckets and objects granted through any access control lists (ACLs)

Além disso, vamos precisar criar uma policy para garantir acesso anônimo para que os usuários possam acessar o bucket. Ainda na aba Permissions, vá em Bucket policy > Edit. Cole o conteúdo abaixo, substituindo a tag <NOME_DO_SEU_BUCKET> pelo nome do seu bucket:

                        <code>
                        {
                           "Version": "2023-04-17",
                           "Statement": [
                              {
                                 "Effect": "Allow",
                                 "Principal": "*",
                                 "Action": [
                                    "s3:GetObject"
                                 ],
                                 "Resource": [
                                    "arn:aws:s3:::/*"
                                 ]
                              }
                           ]
                        }
                        </code>

Para garantir a segurança do nosso bucket, devemos permitir apenas o acesso ao GetObject para que os usuários possam visualizar a aplicação, mas não possam modificar nenhum conteúdo.

Parabéns! Agora é só comemorar. Acesse novamente o link fornecido em static web hosting e você poderá ver a sua página rodando com sucesso.
